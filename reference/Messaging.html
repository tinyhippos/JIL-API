Widget.Messaging
class Messaging
extends Object


Messaging is the container object for JIL Messaging objects.

Messaging is a sub-object of Widget. So we use Messaging in this way: Widget.Messaging.XXX

findMessages

void findMessages(<Message> comparisonMsg, <String> folderType, <Number> startInx, <Number> endInx)
    This method search messages with designed condition in specified message box. This is an asynchronous function, and will invoke Messaging.onMessagesFound() when search completes.

    e.g.
        var msg = Widget.Messaging.createMessage(Widget.Messaging.MessageTypes.SMSMessage);
        msg.addAddress("destination","10086");
        Widget.Messaging.onMessagesFound = myCallBack;
        Widget.Messaging.findMessages(msg, Widget.Messaging.MessageFolderTypes.INBOX, 0, 9);
        function myCallBack(messages) {
            var size = messages.length;
        }
    Parameters:
      comparisonMsg - the comparison messaqge which defines the rules for finding.
      folderType - the name of the folder. Possible values are "inbox", "outbox", "sentbox", "drafts" and "all".
      startInx - an integer value to specify the index the returning search results starts from
      endInx - an integer value to specify the index to which the returning search results ends at

onMessagesFound

void onMessagesFound(<Array> messagesFound, <String> folderType)
    Callback method to invoke when Messaging.findMessages completes.
    Called asynchronously when the message send to the recipient failed due to mail format error or mail/smsc server rejected or network error.
    e.g.
        var msg = Widget.Messaging.createMessage(Widget.Messaging.MessageTypes.SMSMessage);
        msg.addAddress("destination","10086");
        Widget.Messaging.onMessagesFound = myCallBack;
        Widget.Messaging.findMessages(msg, Widget.Messaging.MessageFolderTypes.INBOX, 0, 9);
        function myCallBack(messages) {
            alert("matched message number: "+ messages.length);
        }
    Parameters:
      messagesFound - An array of Message objects, null value will be returned if no matched messages
      folderType - the name of the folder.
