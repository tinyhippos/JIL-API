processorUtilizationPercent

String processorUtilizationPercent
    This is the percent utilization of the phone's processor. E.g., if the processor has a 20% load (80% idle) then the value will be 20.

        }
    Parameters:

onPositionRetrieved

void onPositionRetrieved(<Positioninfo> locationInfo, <String> method)
    This is a callback function that is invoked as a response to the requestLocationInfo() method. No object will be returned if the location could not be identified.

    e.g.
        Widget.Device.DeviceStateInfo.onPositionRetrieved = myCallback;
        function myCallback(locationInfo, method) {
        }
    Parameters:
      locationInfo - The location info.
      method - specifies how the position information has been obtained, possible values are "cellid", "gps" and "agps".

onScreenChangeDimensions

void onScreenChangeDimensions(<Number> newWidth, <Number> newHeight)
    This callback function is invoked when the phone's display changes characteristics. E.g., this would be called if the phone were to switch from landscape to portrait mode.

    e.g.
        Widget.Device.DeviceStateInfo.onScreenChangeDimensions = myCallback;
        function myCallback(newWidth, newHeight) {
        }
    Parameters:
      newWidth - The new screen width. This field should be an integer value.
      newHeight - The new screen height. This field should be an integer value.

requestPositionInfo

void requestPositionInfo(<String> method)
    This issues an asynchronous request for the phone's location.
    This is an asynchronous call. The widget must register with the onPositionRetrieved callback method in order to receive the position information once it is available.

    e.g.
        Widget.Device.DeviceStateInfo.requestPositionInfo();
    Parameters:
      method - specifies how to get the position information, possible values are "cellid", "gps" and "agps"
