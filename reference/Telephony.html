Widget.Telephony
class Telephony
extends Object


Telephony is the container object for JIL Telephony objects.

Telephony is a sub-object of Widget. So we use Telephony in this way: Widget.Telephony.XXX

findCallRecords

void findCallRecords(<CallRecord> comparisonRecord, <Number> startInx, <Number> endInx)
    Retrieves all call records of the specified call record type having a match against the specified fields defined in the comparisonRecord. Supports * as wildcards in Strings. Search is case insensitive.

    Widget can specify only return a range of matched files between startInx and endInx.

    e.g.
        Widget.Telephony.onCallRecordsFound = myCallBack;
        var record = new CallRecord();
        record.callRecordAddress = "55555555";
        Widget.Telephony.findCallRecords(record, 0, 9);     function myCallBack(callRecords) {
        }
    Parameters:
      comparisonRecord - a CallRecord object containing fields to match against in the call records. Leave a field blank if you do not wish to check against it. If the CallRecordType field is blank, then all CallRecord types are searched. Wildcards are supported in the name strings and address strings. To search for records in a given time/date range, define the start time for the search in the startTime value of the comparisonRecord and set the durationSeconds to reflect the end time for the search.
      startInx - an integer value to specify the index the returning search results starts from
      endInx - an integer value to specify the index to which the returning search results ends at

onCallEvent

void onCallEvent(<String> callType, <String> phoneNumber)
    This is a call back method which will be triggered when there's an received call, missed call or outgoing call.

    e.g.
        Widget.Telephony.onCallEvent = myCallBack;
        function myCallBack(callType, phoneNumber) {
        if(callType == "missed") {
            var missedCall = phoneNumber;
        }
    Parameters:
      callType - identifies the type of the call event, possible values are "received", "missed" and "outgoing"
      phoneNumber - if this is an received or missed call, then this is the destination phone number; if this is an outgoing call, then this is the source phone number.

onCallRecordsFound

void onCallRecordsFound(callRecords)
    This is a call back method to invoke when Telephony.findCallRecords completes.

    e.g.
        Widget.Telephony.onCallRecordsFound = myCallBack;
        var record = new CallRecord();
        record.callRecordAddress = "55555555";
        Widget.Telephony.findCallRecords(record, 0, 9);     function myCallBack(callRecords) {
        }
    Parameters:
      Array - callRecords An array of CallRecord objects if either startInx or endInx is negative in Telephony.findCallRecords function, the returned callRecordsFound will be an empty array. if startInx equals endInx in Telephony.findCallRecords function, the returned callRecordsFound will contain only one item whose sequence number is startInx. if startInx is greater than endInx in Telephony.findCallRecords function, the returned callRecordsFound will be an empty array. if startInx is greater than the number of found items in Telephony.findCallRecords function, the returned callRecordsFound will be an empty array. if endInx is greater than the number of found items in Telephony.findCallRecords function, the returned callRecordsFound will contain items bwteeen startInx and the last returned item inclusively.
